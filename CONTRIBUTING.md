# Как внести вклад в TintoraAI

Привет! Спасибо, что хочешь помочь с проектом TintoraAI! Здесь мы расскажем, как ты можешь внести свой вклад, даже если ты новичок. Следуй этим шагам, и всё будет просто!

## Шаги для участия

### 1. Прочитай этот файл
Убедись, что ты понимаешь, как работает проект, и что ты хочешь сделать (например, исправить ошибку или добавить новую функцию).

### 2. Создай учётную запись на GitHub
Если у тебя её ещё нет, зарегистрируйся на [github.com](https://github.com). Это бесплатно!

### 3. Найди задачу
- Посмотри раздел "Issues" в репозитории [TintoraAi](https://github.com/T1desLuck/TintoraAi/issues).
- Выбери задачу, которая помечена как "help wanted" или просто интересна тебе.
- Если не нашёл, создай новый "Issue" и опиши, что хочешь сделать.

### 4. Согласуй задачу
- Напиши комментарий в "Issue" или создай новый, чтобы обсудить идею с автором (T1desLuck).
- Подожди ответа, чтобы убедиться, что твоя идея подходит.

### 5. Настрой среду разработки
Следуй инструкциям в `INSTALL.md`, чтобы установить всё необходимое на своём компьютере.

### 6. Создай ветку
- Открой терминал и введи:
  ```bash
  git clone https://github.com/T1desLuck/TintoraAi.git
  cd TintoraAi
  git checkout -b моя-новая-ветка
  ```
  (Замени `моя-новая-ветка` на своё название, например, `fix-bug-123`.)

### 7. Сделай изменения
- Отредактируй код в папке `src/` или добавь новые файлы.
- Убедись, что код соответствует правилам (проверь с помощью `flake8 src/ tests/`).

### 8. Проверь свой код
- Запусти тесты локально:
  ```bash
  pytest tests/
  ```
- Убедись, что всё работает без ошибок.

### 9. Зафиксируй изменения
- Сохрани изменения командой:
  ```bash
  git add .
  git commit -m "Исправил баг #123: описание изменений"
  ```
  (Замени `#123` на номер задачи из "Issues".)

### 10. Отправь изменения
- Отправь свою ветку на GitHub:
  ```bash
  git push origin моя-новая-ветка
  ```

### 11. Создай Pull Request (PR)
- На сайте GitHub нажми "Create Pull Request".
- Опиши, что ты сделал, и свяжи PR с задачей из "Issues".
- Жди, пока автор проверит и одобрит твои изменения.

## Правила и советы
- Пиши код на Python 3.8.
- Следуй стилю PEP 8 (максимум 88 символов в строке, смотри `.flake8`).
- Добавляй тесты для новых функций в папку `tests/`.
- Будь вежлив в комментариях и готов к правкам.

## Нужна помощь?
Если что-то непонятно, напиши в "Issues" или спроси автора напрямую. Мы рады помочь новичкам!

Спасибо за вклад! 🚀
